# Решение проблем с кодировкой и компиляцией Pawn проектов

## Почему появляются кракозябры (䫿 譴ඨ蠮)?

Проблема возникает из-за несоответствия кодировок:
1. **Исходная кодировка**: Большинство редакторов для Pawn (Sublime Text, Notepad++, старые IDE) сохраняют файлы в кодировке **Windows-1251** (ANSI).
2. **Ожидаемая кодировка**: Современные компиляторы и веб-интерфейсы часто ожидают **UTF-8**.
3. **Результат**: При чтении файла в неправильной кодировке русские символы превращаются в нечитаемые знаки, а комментарии ломают структуру кода, вызывая ошибки `unbalanced braces` и `expected ')'`.

## Как исправить?

### Способ 1: Автоматический скрипт (Рекомендуется)
В корне проекта создан скрипт `fix_encoding.sh`. Он автоматически найдет все файлы `.pwn`, `.inc` и `.cfg` и перекодирует их из Windows-1251 в UTF-8.

**Инструкция:**
1. Откройте терминал в папке с проектом.
2. Запустите команду:
   ```bash
   chmod +x fix_encoding.sh
   ./fix_encoding.sh
   ```
   *(Если проект в подпапке: `./fix_encoding.sh path/to/project`)*
3. После завершения попробуйте скомпилировать проект снова.

### Способ 2: Ручное исправление в редакторе
Если вы используете локальный редактор:
1. Откройте файл `.pwn`.
2. Найдите меню "Encoding" или "Кодировка".
3. Выберите **"Convert to UTF-8"** (или "Save with Encoding -> UTF-8").
   *Важно: Не просто "UTF-8", а именно конвертация, чтобы символы не испортились.*
4. Сохраните файл **без BOM** (Byte Order Mark), если есть такая опция.

## Ошибки компиляции после фикса
Если после исправления кодировки остались ошибки вида:
- `fatal error 030: unbalanced braces`
- `fatal error 031: expected ')', but end of file reached`

Это значит, что из-за кракозябр в комментариях компилятор пропустил реальные скобки `{}` или `()`.
**Решение**: После запуска скрипта `fix_encoding.sh` текст комментариев восстановится, и компилятор сможет корректно прочитать структуру кода.

## Уведомления (Push Notifications)
Теперь при возникновении подобных ошибок система будет выдавать понятное сообщение:
> "Обнаружены символы в неверной кодировке. Вероятно, файл сохранен в Windows-1251. Запустите `fix_encoding.sh` или пересохраните файлы в UTF-8."
